# Primum demo video

3-minute Remotion composition for the hackathon submission. Voice-over is generated by ElevenLabs (or OpenAI TTS as fallback); screen recordings are dropped in as MP4 clips and stitched together with timed captions and an Apollo-themed dark vignette.

The script is in [`src/segments.ts`](src/segments.ts) — a single source of truth used by both the Remotion composition and the voice-over generator.

## Quick start

```bash
# 1 — install
pnpm install

# 2 — generate voiceover (uses ELEVENLABS_API_KEY if set, otherwise OPENAI_API_KEY)
ELEVENLABS_API_KEY=sk_... pnpm gen-voice

# 3 — record screen segments and drop them in public/recordings/
#     (filenames match Visual.src in src/segments.ts)
mkdir -p public/recordings
# … capture the seven clips listed below …

# 4 — preview in Remotion Studio
pnpm dev
# opens http://localhost:3000

# 5 — render the final MP4
pnpm render
# → out/primum-demo.mp4
```

## Voice-over

Defaults to ElevenLabs because their model handles SSML `<break time="…"/>` tags inside the segment text — exactly what this clinical-prose script needs to land its pauses.

| Var | Purpose | Default |
| --- | --- | --- |
| `ELEVENLABS_API_KEY` | preferred provider | required for ElevenLabs |
| `ELEVEN_VOICE_ID` | which Eleven voice | `nPczCjzI2devNBz1zQrb` (Brian, calm narrator) |
| `ELEVEN_MODEL_ID` | which Eleven model | `eleven_turbo_v2_5` |
| `OPENAI_API_KEY` | fallback provider | required if Eleven absent |
| `OPENAI_TTS_MODEL` | which OpenAI model | `gpt-4o-mini-tts` |
| `OPENAI_TTS_VOICE` | which voice | `alloy` |

OpenAI's TTS doesn't accept SSML, so the fallback path strips `<break>` tags and replaces them with commas — close enough for a placeholder, not as crisp as ElevenLabs for the deliberate pauses.

Re-render only specific segments while iterating:

```bash
pnpm gen-voice --segments=01-hook,07-autopilot
```

Dry-run the plan without spending tokens:

```bash
pnpm gen-voice --dry
```

Generated mp3 files land in `public/voiceover/{segment-id}.mp3`. They're gitignored — regenerate locally each time the script changes.

## Editing the script

Open `src/segments.ts`. Each segment is timed in seconds (start + duration), so you don't have to do frame math. The validator at the bottom of the file refuses to compile if segments overlap or overflow the 180-second runtime.

When you change `text` or `ssml`, regenerate just that segment:

```bash
pnpm gen-voice --segments=05-bench
```

## Rendering

```bash
pnpm render          # 1080p MP4 to out/primum-demo.mp4
```

For higher-quality master:

```bash
pnpm render:hi       # PNG image format, lower concurrency, sharper output
```

## Aesthetic

Matches the app's design language: paper-white display type, pink-red severity accents, monospace lower-thirds, soft vignette. The TitleCard uses Source Serif 4 for line 1 and IBM Plex Mono for the optional sub-line — install those locally for the closest match (or `@remotion/google-fonts/SourceSerif4` if you want them bundled into the render).
