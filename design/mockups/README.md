# Mockups

Static HTML visual comps of the three core Primum pages. Open in any browser — no build step.

## Files

- [`index.html`](index.html) — Homepage. Masthead plate, latest cases, corpus callout.
- [`archive.html`](archive.html) — Case archive, chronological back-issue listing.
- [`case-0047.html`](case-0047.html) — Signature case report. The full editorial/clinical treatment: structured abstract, transcript turns with margin notes, severity stamps, pull quote, proposed correction comparison, supervisor consult region, footnotes.

## How to view

```
# From the project root
open primum/design/mockups/index.html
open primum/design/mockups/archive.html
open primum/design/mockups/case-0047.html
```

Or just double-click the files in Finder.

Navigation between the three pages is wired (index → archive → case-0047). External-looking links are placeholders.

## What these prove

- The typographic system (Source Serif 4 + IBM Plex Mono) loads and renders.
- The color tokens (bone + oxblood) work in context.
- The single-column-with-margin-gutter layout survives real content.
- The severity stamp language and margin-note pattern read as intended.
- The editorial voice (case report prose) holds up next to the structural chrome.

## What these are not

- Final content. Every autopsy in production is generated by the pipeline.
- A full component library. Components are rendered inline for simplicity; the Next.js build will factor them.
- Mobile-perfect. The layout collapses reasonably under 900px but the production build will pay more attention to small-screen margin notes.

## Feedback prompts

When reviewing, consider:

1. **Overall tone** — does it feel like a medical journal, an AI product marketing site, or something in between? Target: medical journal. If it leans marketing, we warm or darken.
2. **Body legibility** — is Source Serif 4 at 17px comfortable for long-form reading? Compare against NEJM case reports for density.
3. **Oxblood as accent** — used only for severity, case numbers, footnote markers, hover states. Does it feel clinical or ominous? Adjust saturation in `system.md` if too dark / not dark enough.
4. **Margin notes** — do they feel like scholarly marginalia or like a distracting sidebar? They are the one non-standard layout element.
5. **Severity glyphs** (`○` `◐` `⚠` `✕`) — readable at size? Distinct enough? Too restrained?
6. **Structured abstract block** — bordered block on bone-2 background. Too boxy? Try borderless with a top rule instead?
7. **Pull quote** — full-width top-and-bottom ruled, oxblood italic. Works, or too dramatic?
8. **Supervisor consult** — printed-interview format with Q/A labels. Does it feel like a continuation of the document or like a chat component pretending not to be?
